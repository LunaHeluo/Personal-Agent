# Proposal

## Problem Statement

Luna 在准备投递 AI Agent / AI Product 相关岗位时，需要同时理解 JD、基础简历、
项目经历和求职偏好。当前人工流程分散且耗时，尤其是在判断岗位匹配点、选择要突出的经历、
控制简历表述边界时容易犹豫。她需要一个可重复的求职辅助 Agent，把一次投递前准备整理成
清晰的匹配分析、简历修改建议和投递前判断。

## Product Goal

帮助 Luna 基于基础简历、岗位 JD、个人背景和求职偏好，快速完成针对单个岗位的匹配分析、
简历修改建议、风险提醒和投递前 checklist，并把是否投递、是否修改最终简历等关键动作保留给用户决定。

## Target User

正在求职的 IT master 毕业生 Luna。目标岗位是 Junior AI Agent / AI Product Manager，
地点偏好为上海、深圳、杭州、成都。她的经历包括会议推荐系统、RAG 问答机器人和 TAL AI 产品实习。

## Core Scenarios

1. 用户收到一个 AI Agent 或 AI Product 岗位 JD，Agent 对照基础简历输出岗位匹配分析。
2. 用户准备修改简历，Agent 给出按优先级排序的修改建议，说明应突出产品设计、前端交付、RAG 或实验评估中的哪些经历。
3. 用户准备投递前，Agent 输出风险提醒和 checklist，帮助用户判断投递、暂缓投递或调整岗位方向。

## Non-goals

- 不自动投递岗位、发送邮件或联系 HR。
- 不编造经历、夸大工程深度、模型能力、业务影响或项目 ownership。
- 不替用户决定最终投递方向、最终简历版本或是否记录投递状态。

## Success Metrics

- 一次分析能指出 3 个以上真实匹配点和主要风险。
- 简历修改建议能按优先级排序，并明确对应到 JD 要求。
- 用户能基于 checklist 做出投递、暂缓投递或调整方向的决定。
- 输出能避免捏造和过度包装，保留真实经历边界。

## Risks and Human Approval

主要风险是把产品、前端或 AI 应用经历过度包装成深度后端研发经历，或把课堂、项目、实习成果写成未经验证的业务影响。凡涉及最终投递、最终简历采用、新增或改写经历、调整岗位方向、记录投递状态，都必须由用户确认后执行。
